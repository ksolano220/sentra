# Troubleshooting

Common issues and their fixes.

## SDK returns `Blocked` with risk 100 and no server log

**Cause.** Sentra server is not reachable. The SDK fails closed (blocks by default) when it cannot contact the supervisor.

**Fix.** Confirm the server is running.
```bash
curl http://127.0.0.1:8000/health
# expect: {"status":"ok","risk_threshold":100}
```
If the call fails, start the server: `uvicorn supervisor.main:app --reload`.

---

## SDK returns `Agent Shut Down` on every call

**Cause.** The agent hit the three-strike threshold in a previous session. State persists across server restarts via `supervisor/state_store.json`.

**Fix.** Reset state.
```bash
curl -X POST http://127.0.0.1:8000/reset
```
Or manually delete `supervisor/state_store.json` and `supervisor/runtime_log.json`, then restart the server.

---

## Dashboard shows "No events available"

**Cause.** Neither `supervisor/runtime_log.json` (live) nor `dashboard/demo_log.json` (seed) has content the dashboard can render.

**Fix.** Either run the server and send some actions, or confirm the seed exists.
```bash
ls -l supervisor/runtime_log.json dashboard/demo_log.json
```
If both are missing, pull the latest `dashboard/demo_log.json` from the repo.

---

## Dashboard loads but the Impact Report tab is empty

**Cause.** `dashboard/impact_report.json` is missing. This file is generated by the companion [autonomous-claims-workflow](https://github.com/ksolano220/autonomous-claims-workflow) batch demo, not by Sentra itself.

**Fix.** Run the claims workflow demo and copy its output:
```bash
cd autonomous-claims-workflow
python demo.py
cp outputs/impact_report.json <sentra-repo>/dashboard/
```
The Live Dashboard tab works without this file.

---

## Port 8000 already in use

**Cause.** Another process (or a previous `uvicorn` run) is bound to the port.

**Fix.** Pick a different port and tell the SDK about it.
```bash
uvicorn supervisor.main:app --port 8010 --reload
```
Then in client code:
```python
sentra = Sentra(base_url="http://127.0.0.1:8010")
```

---

## Streamlit dashboard not auto-refreshing

**Cause.** Streamlit's `st.fragment(run_every=...)` requires Streamlit 1.37 or newer.

**Fix.** Upgrade.
```bash
pip install --upgrade streamlit
```
`requirements.txt` already pins a compatible version; reinstall it if you are in a fresh venv.

---

## `/evaluate` endpoint returns `BLOCK` unexpectedly for claims-workflow clients

**Cause.** The `/evaluate` adapter infers `approval_requires_verified_eligibility=True` by default and denies approval notifications when `documents.proof_of_termination` is missing on the claim payload.

**Fix.** Send the full claim shape. See [supervisor/main.py](../supervisor/main.py) `evaluate_claim_action` for the exact contract. For ad-hoc testing without a claim wrapper, hit `/agent-action` directly instead.
